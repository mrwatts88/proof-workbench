# Process

This directory defines how mathematical research moves from an unnormalized idea
to a reviewed proof, disproof, or delimited partial result.

Read in this order:

1. [workflow.md](workflow.md) — the end-to-end operating loop.
2. [statuses.md](statuses.md) — the state model and promotion rules.
3. [proof-standard.md](proof-standard.md) — what counts as a proof.
4. [review-playbook.md](review-playbook.md) — adversarial checking.
5. [records.md](records.md) — persistent files and stable IDs.
6. [toolkit.md](toolkit.md) — mathematical and computational tactics.

Changes to these rules are process decisions. Record significant changes in
`PROJECT_STATE.md` and cover enforceable behavior with tests.
